# AGENT AUTHORITY ROUTING (CLINIC MVP)

Этот документ указывает агентам Antigravity (Gemini), какие файлы читать при работе с локальной системой DENTE.

## Как брать задачи
1. Всегда сверяй статус системы с `dental_agent_system_design.md` (находится в памяти агента или корне проекта).
2. Если задача касается извлечения данных из базы -> читай `docs/DB_DIGGER_MANDATE.md` (если есть) и `database.py`.
3. Если задача касается Яндекс.Карт или WhatsApp-рассылок -> ОБЯЗАТЕЛЬНО читай `docs/MARKETING_SEO_MANDATE.md` перед тем как писать код.
4. Если задача касается телеграм-бота -> смотри `ShadowAnalyst/gui/app.py`.

## Запрещенные действия
- НЕ пытайся подключать внешние SaaS базы данных (Firebase, Supabase, AWS RDS). Только локальный SQLite.
- НЕ запускай сканирование сети (nmap) без явного указания.
- НЕ устанавливай библиотеки через npm (мы не используем React/Node.js, только чистый HTML+Jinja2).

## Роутинг кода (Где что лежит)
- `C:\Clinic_MVP\clinic_admin\` — Панель ресепшена (FastAPI) и генерация задач.
- `C:\Clinic_MVP\ShadowAnalyst\` — Модуль анализа снимков и Telegram интеграция.
- `C:\Clinic_MVP\*.ps1` — Скрипты развертывания сети (Agent-Router).
